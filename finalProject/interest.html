Detail Explanation about Definition

easy development

기업에서 운영하는 웹 서비스에는 비즈니스 로직이라는 것이 있습니다. 비즈니스 로직이란, 기업이 제공하는 서비스를 코드로 구현한 것으로, 사용자의 요구사항을 해결하기 위한 실질적인 코드들을 의미합니다. 스프링이 등장하기 이전에는 비즈니스 로직을 구현하기 위해 기술 자체에 대한 공부를 추가적으로 해야만 했습니다. 그러나, 스프링은 이전 기술에 비해 사용 방법이 상대적으로 덜 복잡합니다. 따라서, 개발 초기에 기본적인 설정과 적용시킬 기술들만 잘 선택을 해준다면, 기술보다는 애플리케이션의 로직 자체에 더 집중하여 비즈니스 로직을 구현할 수 있습니다.

open source

스프링은 모든 사용자에게 무료로 열려있습니다. 즉 어떤 개인 및 기업도 스프링을 사용하여 웹 애플리케이션을 개발할 수 있으며, 필요하다면 스프링의 코드를 일부 수정하여 사용하여도 무관합니다. 스프링은 오픈소스 프레임 워크이지만, 안정적인 개발과 개선이 보장됩니다.

lightweight

기존의 스프링 대신 사용하던 기술들과 비교하여, 스프링을 사용했을 떄에 개발자가 작성해야 할 코드가 상대적으로 단순합니다. 즉, 스프링을 사용함으로써 기존 기술을 사용할 떄에 불가피하게 작성해야만 했던 불필요하게 복잡한 코드를 제거하여 코드의 복잡성을 낮출 수 있습니다.

framework

애플리케이션 프레임워크는 애플리케이션을 개발하는데에 있어 필요한 모든 업무 분야 및 모든 기술과 관련된 코드들의 뼈대를 제공합니다.

advantage of SpringBoot

1

기존에 어려운 초기 설정에 쏟아야 했을 시간과 노력을 절약하여 비즈니스 로직을 구현하는데에 집중할 수 있습니다.

2

스프링부트는 자체적인 웹 서버를 내장하고 있어, 빠르고 간편하게 배포를 진행 할 수 있습니다.

3

스프링부트를 사용하면 독립적으로 실행 가능한 Jar파일로 프로젝트를 빌드할 수 있어, 클라우드 서비스 및 도커와 같은 가상화 환경에 빠르게 배포할 수 있습니다.

POJO

Importance

만약 외부 라이브러리를 import하여 메서드를 사용하고 있는 객체가 있다고 가정해봅시다. 이 객체는 순수 Java외의 기술을 사용하고 있으므로 POJO가 아닙니다. 그런데 이 객체가 사용하고 있는 기술이 중요도가 떨어져 사용이 되지 않거나, 개선된 기술이 등장하여 기존 기술과 관련된 코드를 모두 고쳐야 하는 상황이 발생하면 해당 기술을 사용하고 있는 모든 객체들의 코드를 전부 바꿔 주어야만 합니다.

반면, POJO는 순수 Java만을 사용하여 만든 객체이므로 특정 기술이나 환경에 종속되지 않습니다. 따라서, 외부 기술이나 규약의 변화에 얽매이지 않아, 보다 유연하게 변화와 확장에 대처할 수 있습니다. 이러한 POJO를 사용하여 비즈니스 로직을 구현하면, 객체지향 설계를 제한없이 적용할 수 있으며, 코드가 단순해져 테스트와 디버깅 또한 쉬워집니다.

feature of POJO

IoC / DI

개발자가 작성한 객체 나 메서드의 제어를 개발자가 아니라 외부에 위임하는 설계원칙을 IoC(Inversion of Control)라고 합니다. 즉, 개발자가 아닌 스프링 컨테이너가 필요에 따라 제어를 하는 것입니다.
이러한 제어의 역전과정에서 의존관계를 외부에서 결정해주는 것을 말합니다. 즉 사용할 객체를 스스로 결정하지 않고 스프링 컨테이너가 특정 객체를 결정하여 주입하는 것을 말합니다.

AOP

애플리케이션 개발시 구형해야하는 기능들은 애플리케이션의 핵심기능과 괄련된 관심사항인 핵심 관심 사항과 모든 핵심 관심 사항에 공통적으로 적용되는 관심사항을 의미하는 공통 관심 사항으로 분류할 수 있습니다.
이 떄, 핵심 관심사항과 공통 관심 사항이 코드에 함께 모여 있으면 중복이 발생할 수 밖에 없습니다. 이러한 코드의 중복을 피하기 위해 공통 관심 사항과 관련된 기능들을 별도의 객체로 분리하여야 합니다.
이처럼 애플리케이션 전반에 걸쳐 적용되는 공통 기능을 비즈니스 로직으로 부터 분리 해내는 것을 AOP라고 합니다.

PSA

웹서버는 DB와 소통하며 웹 클라이언트의 요청을 처리하기 때문에 백엔드는 DB와 떼어 놓을 수 없습니다. DB의 종류는 다양합니다. 만약에 특정 DB를 사용하여 개발을 완료하였는데, 다른 DB로 바꿔야 할 때 기존의 방법으로는 매우 복잡한 과정을 거쳐야 합니다. 하지만 스프링은 데이터베이스 서비스를 추상화한 인터페이스(JDBC)를 제공해주기 때문에 더욱 쉽게 DB를 바꿀 수 있습니다.
JDBC처럼 특정 기술과 관련된 서비스를 추상화하여 일관된 방식으로 사용될 수 있도록 한것을 PSA라고 합니다

feature of Spring

POJO

스프링의 가장 큰 특징은 POJO프로그래밍을 지향한다는 것입니다. POJO란, Plain Old Java Object, 즉 순수 Java만을 통해서 생성한 객체를 의미합니다.
순수 Java만을 사용한다는 것은 Java 및 Java의 스펙에 정의된 기술만 사용한다는 의미입니다. 즉, 어떤 객체가 외부의 라이브러리나 외부의 모듈을 가져와서 사용하고 있다면, 그 객체는 POJO라고 할 수 없습니다. POJO는 말 그대로, 다른 기술을 사용하지 않는 순수한 Java만을 사용하여 만든 객체인 것입니다.
